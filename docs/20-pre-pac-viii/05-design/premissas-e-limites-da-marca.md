# Premissas e limites da marca

| Campo | Valor |
|---|---|
| Projeto | Portal Bamak |
| Fase | Pré-PAC VIII |
| Escopo | Área pública, painel administrativo, protótipos e entregáveis do projeto |

| Última atualização | 11/07/2026 |

## 1. Finalidade

Registrar o que foi efetivamente disponibilizado pela Bamak, o que foi padronizado especificamente para o Portal Bamak e quais decisões não podem ser tratadas como normas corporativas oficiais.

Este artefato não substitui um manual de marca. Sua função é preservar a rastreabilidade das decisões e evitar que escolhas de interface sejam apresentadas como definições institucionais amplas da empresa.

## 2. Base disponibilizada pela Bamak

A empresa não forneceu um manual formal de identidade visual.

Os materiais disponíveis permitem confirmar:

| Elemento | Situação |
|---|---|
| Nome BAMAK | presente nos materiais institucionais |
| Símbolo gráfico | presente nos materiais institucionais |
| Assinatura horizontal | presente nos materiais disponíveis |
| Versão verde | presente nos materiais disponíveis |
| Versão branca ou negativa | observada nos materiais disponíveis |
| Verde digital `#00421F` | referência adotada no projeto |
| Manual formal de marca | não fornecido |
| Tipografia corporativa | não identificada |
| Pantone | não informado |
| CMYK | não informado |
| Área de proteção | não informada |
| Tamanho mínimo | não informado |
| Regras fotográficas | não informadas |
| Sistema de interface | não fornecido |

O valor `#00421F` deve ser descrito como:

> Verde digital de referência adotado no Portal Bamak a partir dos materiais disponíveis.

Ele não deve ser apresentado como especificação universal e oficial da empresa para todos os meios.

## 3. Classificação das decisões

As decisões visuais devem ser registradas conforme seu grau de autoridade.

| Classificação | Significado |
|---|---|
| `fornecido pela Bamak` | recebido diretamente ou observado em materiais disponibilizados pela empresa |
| `referência adotada no projeto` | valor extraído ou consolidado para o Portal Bamak |
| `decisão operacional do projeto` | escolha criada para viabilizar a interface e sua implementação |
| `pendente de validação` | decisão ainda não apresentada ou confirmada pela empresa |
| `não definido` | regra que não pode ser inferida com segurança |

## 4. Decisões operacionais do projeto

As decisões abaixo pertencem ao Portal Bamak e não devem ser apresentadas como regras corporativas gerais.

| Decisão | Classificação |
|---|---|
| assinatura empilhada | decisão operacional do projeto |
| biblioteca em AI, SVG e PNG | padronização técnica do projeto |
| verde `#00421F` | referência digital adotada |
| Archivo e Source Sans 3 | tipografia da interface |
| escalas `brand`, `neutral` e feedback | paleta operacional da interface |
| tokens semânticos | estrutura técnica do produto |
| Lucide | iconografia da interface |
| grid de 4, 8 e 12 colunas | decisão de layout |
| raios, bordas e elevação | decisão de UI |
| linguagem fotográfica | direção visual do portal |
| movimento e transições | decisão de interação |
| diferenças entre área pública e painel | decisão do produto digital |

A assinatura empilhada foi construída com os elementos vetoriais existentes. Ela não deve ser descrita como assinatura institucional oficial sem confirmação explícita da empresa.

## 5. Elementos não definidos

Não há base suficiente para afirmar como oficiais:

- tipografia corporativa;
- paleta corporativa completa;
- assinatura empilhada;
- área de proteção;
- tamanho mínimo;
- Pantone;
- CMYK;
- gradientes institucionais;
- regras de fotografia corporativa;
- biblioteca de ícones da empresa;
- aplicações em uniformes, veículos, fachadas ou embalagens;
- design system corporativo;
- situação jurídica ou registral da marca.

Esses pontos permanecem como `não definidos` até que a Bamak forneça orientação formal.

## 6. Limites de afirmação

Evitar estas formulações:

```text
manual de marca da Bamak
identidade visual oficial
fonte oficial
paleta oficial completa
novo logotipo da Bamak
design system oficial da empresa
```

Usar:

```text
base visual do Portal Bamak
padronização operacional do projeto
referência digital adotada
composição preparada para o portal
direção visual preliminar
decisão de interface
validada para uso no portal
```

Os artefatos do projeto devem distinguir claramente entre:

- elementos fornecidos pela empresa;
- referências adotadas;
- decisões de UX/UI;
- decisões ainda pendentes.

## 7. Escopo de aplicação

As decisões documentadas são válidas inicialmente para:

- área pública do Portal Bamak;
- painel administrativo;
- protótipos;
- style tile;
- documentação acadêmica;
- apresentações do projeto;
- entregáveis do PAC e do portfólio;
- implementação do portal.

Não devem ser aplicadas automaticamente a:

- materiais comerciais gerais;
- redes sociais;
- impressos institucionais;
- uniformes;
- fachadas;
- veículos;
- embalagens;
- outros produtos digitais;
- campanhas externas da empresa.

O uso fora do Portal Bamak exige alinhamento específico com a Bamak.

## 8. Interpretação da validação da empresa

A aprovação do style tile, de um protótipo ou de uma tela significa:

> A direção foi considerada adequada para o Portal Bamak naquele estágio.

Essa aprovação não significa automaticamente:

> A empresa adotou essas decisões como manual corporativo para todos os canais.

As respostas da empresa devem ser registradas conforme seu efeito.

| Resposta | Efeito |
|---|---|
| aprovada para o portal | a decisão pode avançar no produto |
| alteração solicitada | os artefatos e protótipos devem ser atualizados |
| adotada como regra institucional | somente quando a empresa declarar isso explicitamente |

## 9. Hierarquia de autoridade

Quando houver divergência, deve ser usada esta ordem:

1. manual ou norma formal futura fornecida pela Bamak;
2. orientação específica e registrada pela empresa para o Portal Bamak;
3. assets e materiais originais fornecidos pela empresa;
4. decisões operacionais documentadas no projeto;
5. experimentações ainda não validadas.

Uma aprovação pontual do portal não substitui uma norma formal fornecida posteriormente.

## 10. Recebimento de normas futuras

Caso a Bamak forneça futuramente um manual, norma ou orientação formal:

1. comparar o novo material com os assets e decisões existentes;
2. identificar conflitos;
3. priorizar as normas formais da empresa;
4. atualizar o arquivo mestre da marca, quando autorizado;
5. ajustar paleta, tipografia, iconografia e componentes afetados;
6. revisar os artefatos de fundamentos;
7. revisar o style tile e os protótipos;
8. registrar a alteração no histórico do projeto;
9. preservar a rastreabilidade da decisão anterior.

Nenhuma decisão anterior deve ser removida sem registro da mudança.
